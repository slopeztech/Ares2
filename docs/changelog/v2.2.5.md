# v2.2.5 — Safety: `formatHkFieldValueLocked` returns false on sensor failures; "nan" sentinel in CSV (H4)

**Date:** 2026-05-18  
**Type:** Patch  
**Branch:** `feature/h4-sensor-nan-sentinel`

---

## Summary

Fixes [H4]: `formatHkFieldValueLocked` previously returned `true` even when
a sensor driver returned an error (e.g. `BaroStatus::ERROR`), masking the
failure from all callers.  The function now returns `false` for any failure
path (driver fault or invalid VAR entry) while continuing to write the `"nan"`
error sentinel into the CSV column.  Callers are simplified — they always emit
the value string whether it is a real reading or the sentinel.
