# ARES v2.2.6 — Changelog

**Released:** 2026-05-19  
**Previous version:** 2.2.5  
**Type:** Patch — static analysis compliance with one REST error-code behavior change.

## Summary

Resolves all findings reported by `ares_code_lint` (1 ERROR + 39 WARNINGs)
across 13 source files.  Most changes are documentation and compliance-only,
but `/api/pulse/status` now returns **HTTP 500** instead of **HTTP 503** for
the subsystem-unavailable condition (REST-2.3 compliance).  Downstream REST
clients that specifically handled 503 on this endpoint should be updated.
All 137 native unit tests pass unchanged.

## Notes on CERT-14.3 (`static` local arrays)

All array declarations promoted to `static` belong to functions that are:

- **AMS engine methods** (`*Locked` suffix) — always called while `mutex_` is held; effectively single-threaded.
- **API handler methods** — the REST server processes one request at a time (documented in `api_server.cpp` line 54–57: *"The API server is single-threaded (one request at a time)"*).

In both contexts there is no re-entrancy, so function-local `static` storage is safe.

## Notes on CERT-16.1 (float comparison)

`std::numeric_limits<float>::min()` equals `FLT_MIN` ≈ 1.175 × 10⁻³⁸ (minimum
positive normalized `float`).  The new guard rejects zero and subnormal values as
divisors — a stricter and safer criterion than exact zero equality.
