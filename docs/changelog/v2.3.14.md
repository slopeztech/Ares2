# ARES v2.3.14 ŌĆö Changelog

**Released:** 2026-05-31
**Previous version:** 2.3.13
**Type:** Patch ŌĆö checkpoint timing markers committed only after confirmed write; 1 new SITL retry test.

---

## Summary

This release hardens checkpoint persistence robustness in the AMS engine.
Previously, `saveResumePointLocked()` advanced `lastCheckpointMs_` and cleared
`checkpointDirty_` before `flushPendingIoUnlocked()` had a chance to call
`writeFile()`.  If the write failed (e.g. flash full), the engine treated the
checkpoint as successfully persisted and deferred the next attempt by a full
`AMS_CHECKPOINT_INTERVAL_MS`, leaving the on-disk checkpoint stale indefinitely
until the next material state change.

After this fix, `lastCheckpointMs_` and `checkpointDirty_` are updated only
inside `flushPendingIoUnlocked()`, after `writeFile()` returns `OK`.  On
failure the dirty flag is re-armed and `lastCheckpointMs_` is left at the last
confirmed value, so the engine retries on the very next tick once the interval
has elapsed.

---

## Changes

**Checkpoint timing markers deferred to confirmed flush**

`mission_script_engine_internal.h` ŌĆö `PendingCheckpoint` gains a `uint64_t nowMs`
field to carry the staging timestamp through to the flush step.

`mission_script_engine_checkpoint.cpp ŌĆö saveResumePointLocked()` ŌĆö Removed the
`lastCheckpointMs_ = nowMs` and `checkpointDirty_ = false` assignments.
Instead, `nowMs` is stored in `pendingCheckpoint_.nowMs`.

`mission_script_engine_checkpoint.cpp ŌĆö flushPendingIoUnlocked()` ŌĆö After
`writeFile()` returns:
- **Success**: re-acquires the mutex and sets `lastCheckpointMs_` and
  `checkpointDirty_ = false`.
- **Failure**: re-acquires the mutex and re-arms `checkpointDirty_ = true`
  without touching `lastCheckpointMs_`, guaranteeing a retry on the next
  eligible tick.

**New SITL test**

`test/test_ams_integration/test_ams_checkpoint_flush.cpp` ŌĆö Added
`test_checkpoint_retried_after_write_failure`: injects one `NO_SPACE` failure
via `failNextWrites(1)`, verifies the attempt was made, advances
`AMS_CHECKPOINT_INTERVAL_MS + 1 ms`, and confirms the retry succeeds with the
correct state record.

`test/stubs/hal_sim/sim_storage_driver.h` ŌĆö Added `failNextWrites(uint8_t count)`
helper (analogous to the existing `failNextAppends`) and a `failWriteCount_`
injection counter checked at the start of `writeFile()`.
