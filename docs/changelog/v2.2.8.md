# ARES v2.2.8 — Changelog

**Released:** 2026-05-19  
**Previous version:** 2.2.7  
**Type:** Patch — fix: replace hardcoded telemetry-interval literals with config.h constants ([M2]).

---

## Summary

Resolves todo item **[M2]**: `radio_dispatcher.cpp` hardcoded the bounds `[100, 60000]`
for the `SET_TELEM_INTERVAL` command instead of reading `ares::TELEMETRY_INTERVAL_MIN`
and `ares::TELEMETRY_INTERVAL_MAX` from `config.h`.  If those constants are ever
adjusted, the dispatcher would silently enforce the old limits — a maintenance hazard.

The fix replaces both literals in the range-check condition and the log message,
and adds three new boundary unit tests that pin the expected behaviour at the exact
minimum, maximum, and one-above-maximum values using the same constants.

## Changes

### `src/comms/radio_dispatcher.cpp`

| Location | Change |
|----------|--------|
| `SET_TELEM_INTERVAL` handler (~line 549) | Range-check condition: `intervalMs < 100U \|\| intervalMs > 60000U` → `intervalMs < ares::TELEMETRY_INTERVAL_MIN \|\| intervalMs > ares::TELEMETRY_INTERVAL_MAX` |
| Same handler | Inline comment: `"Sanity bounds: 100 ms .. 60 000 ms."` → `"Sanity bounds: [TELEMETRY_INTERVAL_MIN, TELEMETRY_INTERVAL_MAX] (config.h)."` |
| Same handler | `LOG_W` format string: `"out of range [100, 60000]"` → `"out of range [%u, %u]"` with `ares::TELEMETRY_INTERVAL_MIN` / `ares::TELEMETRY_INTERVAL_MAX` as arguments |

### `test/test_dispatcher/test_command_dispatch.cpp` — 3 new tests

| Test | Interval | Expected result |
|------|----------|-----------------|
| `test_cmd_nonfrag_set_telem_interval_at_min` | `TELEMETRY_INTERVAL_MIN` (100 ms) | Passes range check; `EXECUTION_ERROR` (no active HK slots) |
| `test_cmd_nonfrag_set_telem_interval_at_max` | `TELEMETRY_INTERVAL_MAX` (60 000 ms) | Passes range check; `EXECUTION_ERROR` (no active HK slots) |
| `test_cmd_nonfrag_set_telem_interval_above_max` | `TELEMETRY_INTERVAL_MAX + 1` (60 001 ms) | `INVALID_PARAM` (rejected by range check) |

### `test/test_dispatcher/main.cpp`

- Test count updated: 50 → 53.
- Three new `extern` declarations and `RUN_TEST` calls added.
