## Summary

Resolves finding **[H2]**: `ares_log.cpp` had 0 % line coverage because the
existing three `test_log_api` tests only exercised `setLevel`/`getLevel` and a

## Root cause (H2)

`ares_log.cpp` contains three distinct defensive paths inside `emitV()` that
were never reached by any test:

| Path | Line(s) | Condition |
|------|---------|-----------|
| Null-pointer guard | 38ÔÇô41 | `tag == nullptr \|\| fmt == nullptr` Ôćĺ early return |
| Header-overflow clamp | 52ÔÇô55 | `snprintf` returns < 0 or Ôëą `LOG_BUF_SIZE` |
| Message-truncation clamp | 73ÔÇô76 | `pos + msgLen Ôëą LOG_BUF_SIZE Ôłĺ 1` Ôćĺ cap + write `'\n'` |

A 200-character message body exceeds `LOG_BUF_SIZE` (192 bytes) once the
`[uptime] L TAG: ` header is prepended, forcing the truncation branch.
