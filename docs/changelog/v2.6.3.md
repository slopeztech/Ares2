# ARES v2.6.3 - Changelog

Previous version: 2.6.2
Type: Patch - centralized installed-driver registry, runtime driver selection defaults, and docs/examples alignment.

---

## Summary

This patch consolidates driver model management into a single source of truth and extends runtime default-driver selection.

- Added a centralized installed-driver registry consumed by runtime, API, AMS aliases, and config validation.
- Added runtime selectors for GPS, BARO, and COM (IMU selector preserved and aligned to shared interface).
- Extended device configuration to persist and validate all default driver models:
  - `default_gps_driver`
  - `default_baro_driver`
  - `default_com_driver`
  - `default_imu_driver`
- Updated `PUT /api/device/config` to apply all runtime selectors after validation/persist.
- Updated hardware/API docs and added a new AMS example for accelerometer up/down transitions using ADXL375.
- Cleared static-analysis regressions introduced during refactor (clang-tidy/cppcheck practical/MISRA pipeline clean).

---

## Functional changes

1. Central installed-driver registry

- New registry in `src/sys/hardware/installed_driver_registry.*` with typed lists per driver kind.
- Registry exposes dynamic model validation and canonical defaults used by config/API/runtime.
- AMS registry aliases were aligned to use centralized types.

2. Runtime selectors for all core sensor/comm groups

- Added generic `DriverSelectionInterface` and integrated selectors for:
  - `src/sys/gps/gps_selector.*`
  - `src/sys/baro/baro_selector.*`
  - `src/sys/com/com_selector.*`
  - `src/sys/imu/imu_selector.*` (aligned)
- `ApiServer` and `main` wiring now pass/select all defaults at runtime.

3. Device config schema and API application path

- DeviceConfig schema updated to include persisted defaults for GPS/BARO/COM/IMU.
- Validation now checks model support dynamically against the installed-driver registry.
- `PUT /api/device/config` now applies all four selectors after successful persist.

---

## Docs and examples

- API documentation updated with full `default_*_driver` examples and dynamic model table from registry.
- Hardware docs updated to reflect runtime-selectable IMU and current bus mapping notes.
