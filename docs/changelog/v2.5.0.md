# ARES v2.5.0 — Changelog

**Released:** 2026-06-02
**Previous version:** 2.4.1
**Type:** Minor — P1-7 narrow bare `except Exception` in scripts; A2-6 obsolete example warning.

---

## Summary

This patch closes two long-standing quality findings in the support toolchain:

- **P1-7** — Four bare `except Exception:` clauses in `scripts/` silently
  swallowed unrelated errors (e.g. a `KeyError` or `AttributeError` in
  the UI rendering path).  Each clause is now narrowed to the precise
  exception types that can realistically occur at that call site.

- **A2-6** — `ams_examples/old/test.ams` contained `include LORA as COM`,
  referencing a model name (`"LORA"`) that is not present in the
  compiled-in driver registry.  The script would always fail to parse
  with *"model 'LORA' not found in compiled-in drivers"* and was
  misleading for anyone using the `old/` folder as a reference.
  A prominent `NOTE` comment has been added explaining the issue and
  pointing to the correct model (`DXLR03`).

Firmware changes include AMS-4.20 BUZZER.beep support (HAL interface, drivers,
engine parser/runtime, and SITL tests).  The master CI pipeline continues to
report **Pass = 7 / Warn = 0 / Fail = 0 (OVERALL PASS)**.

## Changes

### 1. `scripts/ares_api_tui.py`

Three `except Exception:` clauses narrowed:

| Location | Old | New | Rationale |
|---|---|---|---|
| `current_wifi_ssid()` subprocess call | `except Exception:` | `except (subprocess.TimeoutExpired, OSError):` | Only `netsh` invocation can fail here; `TimeoutExpired` covers the `timeout=3` limit, `OSError` covers missing executable on non-Windows |
| `int(rec_err)` conversion in status panel | `except Exception:` | `except (ValueError, TypeError):` | `int()` raises exactly these two types on bad input |
| `json.loads(text)` pretty-print | `except Exception:` | `except (ValueError, UnicodeDecodeError):` | JSON decode raises `ValueError`; the upstream `decode()` with `errors="replace"` is already guarded, but a subsequent re-encode could raise `UnicodeDecodeError` |

### 2. `scripts/ares_fs_tui.py`

| Location | Old | New | Rationale |
|---|---|---|---|
| `current_wifi_ssid()` subprocess call | `except Exception:` | `except (subprocess.TimeoutExpired, OSError):` | Identical pattern to `ares_api_tui.py` |

### 3. `scripts/rx_test.py`

- Added `from textual.css.query import NoMatches` to the Textual import block.
- `update_layout_for_size()` `except Exception: pass` narrowed to
  `except NoMatches: pass`.  `query_one()` raises exactly `NoMatches`
  when a widget is not yet in the DOM; any other exception in the
  layout path should propagate to the Textual error handler.

### 4. `ams_examples/old/test.ams`

Added a four-line `NOTE (A2-6)` block at the top of the file explaining
that the script is non-functional as written because `"LORA"` is not a
registered driver model, and directing the reader to use `DXLR03`
instead.
