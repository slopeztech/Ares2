# ARES v2.6.2 - Changelog

Previous version: 2.6.1
Type: Patch - AMS calibration bootstrap fix and runtime telemetry hardening.

---

## Summary

This patch fixes a runtime issue where `CALIBRATE(...)` variables could remain invalid (`nan`) when calibration started before `arm()`, and hardens integer runtime field mapping in HK payload paths.

- Fixed AMS tick behavior so pending `CALIBRATE` actions can advance while the mission is loaded but not yet execution-enabled.
- Preserved pre-arm safety: transitions, periodic actions, and task execution remain disabled until `arm()`.
- Avoided float round-trip for integer-backed `RUNTIME` fields mapped to APUS HK payload slots.
- Updated docs/examples consistency for current release version references.

---

## Runtime fix details

1. Pre-arm calibration progression

- `stepPendingCalibrationsLocked(...)` now advances in pre-arm bootstrap (`status = LOADED`, `executionEnabled = false`).
- This allows `set zero_alt = CALIBRATE(BARO.alt, N)` to complete before entering armed execution.
- Transition logic is still gated by `executionEnabled`.

2. Integer-safe HK runtime mapping

- Direct mapping is used for integer protocol fields:
  - `RUNTIME.uptime_ms` -> `timestampMs`
  - `RUNTIME.state_idx` -> `flightPhase`
  - `RUNTIME.status_bits` -> `statusBits`
- This removes precision-loss risk from float conversions on long uptimes.
