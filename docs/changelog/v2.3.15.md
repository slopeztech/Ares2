# ARES v2.3.15 — Changelog

**Released:** 2026-06-01
**Previous version:** 2.3.14
**Type:** Patch — hardening, defensive coding, parser strict-mode, sensor NaN guard, BMP280 test suite, clang-tidy clean, coverage uplift.

---

## Summary

This release is a broad hardening pass across the AMS engine, API layer, sensor
drivers, and radio dispatcher.  The main themes are:

- **Checkpoint build atomicity** — `buildCheckpointRecordLocked` and its three
  append helpers now propagate write failures, aborting the record rather than
  silently emitting a truncated or corrupt checkpoint.
- **Parser trailing-garbage rejection** — Six `sscanf`-based parsers now use
  `%n` position tracking to detect and reject unexpected tokens after
  well-formed directives.
- **`radio.config` refactor** — `parseRadioConfigLineLocked` replaced ad-hoc
  `sscanf` with `TokenCursor` + `parseFloatValue`, gaining NaN/Inf rejection
  and dotted-key support.
- **Sensor NaN guard** — GPS, baro, and IMU field readers now invalidate their
  cache entry if the returned value is non-finite.
- **`memset` → `std::fill_n`** — Type-safe initialisation for `uint64_t` and
  `bool` arrays throughout the engine.
- **API defensive returns** — All `ARES_ASSERT` calls in the API and storage
  handlers replaced with graceful HTTP 500 responses.
- **BMP280 test suite** — New `test_bmp280` suite added to `sim` and `coverage`
  environments; `Wire.h` stub upgraded to a full virtual-method class.
- **v4 strict-restore regression tests** — Four new tests covering wrong
  varCount, wrong hkSlotCount, and truncated confirm/hold sections.
- **clang-tidy PASS** — Two `readability-function-size` suppressions applied;
  pipeline goes from WARN to PASS.
- **Coverage uplift** — `checkpoint.cpp` 65.7 % → 73.8 %, `parser_actions.cpp`
  69.2 % → 70.1 %; overall line coverage 77.5 % → 78.3 %.

After this release the master CI pipeline reports **Pass = 7 / Warn = 0 / Fail = 0
(OVERALL PASS)**.

## Changes

### 1. Checkpoint build atomicity (`src/ams/mission_script_engine_checkpoint.cpp`)

`appendVarsSectionLocked`, `appendSlotTimersSectionLocked`, and
`appendConfirmHoldSectionLocked` changed return type from `void` to `bool`.
Each `snprintf` call now captures the remaining buffer space into a local
`rem` variable and returns `false` if the write would overflow.

`buildCheckpointRecordLocked` propagates these failures: if any section
returns `false` a `LOG_W` is emitted and the whole record is discarded,
preventing a partial checkpoint from reaching the flash.

Declaration signatures updated in **`src/ams/mission_script_engine.h`**
accordingly.

`restoreCheckpointV4StrictLocked` extracted from `restoreCheckpointV4FieldsLocked`
and declared in the header.  The strict path validates varCount and
hkSlotCount against the loaded program before applying any state.

Checkpoint body format: when `program_.varCount == 0` the vars section
writes nothing (no `varCount` field in the record).  Test vectors in
`test_ams_script_variants.cpp` and the restore logic updated to match.

**`src/ams/mission_script_engine.h`** — `condIdx` parameter of
`evaluateOneTransitionConditionLocked` changed from `uint8_t` to `size_t`.

### 2. Runtime index safety (`src/ams/mission_script_engine_runtime.cpp`)

`prevBase` and `flatIdx` in `evaluateTransitionAndMaybeEnterLocked` changed
from `uint8_t` to `size_t` to prevent unsigned wrapping when
`AMS_MAX_TRANSITIONS × AMS_MAX_TRANSITION_CONDS` exceeds 255.  A bounds
check `flatIdx < std::size(transitionPrevVal_)` added before the array
access.

`kPrevSlots` in `enterStateLocked` changed from `uint8_t` to `size_t` for
the same reason.

### 3. Parser trailing-garbage rejection (`src/ams/mission_script_engine_parser_actions.cpp`)

Added `%n` position capture to `sscanf` calls in:

- `parseOnErrorEventLineLocked` — rejects tokens after `EVENT verb "text"`.
- `parseOnExitEventLineLocked` — rejects tokens after `EVENT verb "text"`.
- `parseOnTimeoutEventLineLocked` — rejects tokens after `EVENT verb "text"`.
- `parseOnTimeoutTransitionLineLocked` — rejects tokens after `transition to TARGET`.
- `parseDeltaSetActionLocked` — rejects tokens after `EXPR delta`.

### 4. Parser name-length detection

**`src/ams/mission_script_engine_parser_cond.cpp`** — `parseTransitionLineLocked`
and `parseFallbackTransitionLineLocked` now detect target state names longer
than 15 characters (previously silently truncated by `%15s`).

**`src/ams/mission_script_engine_parser_finalize.cpp`** — `parseTaskLineLocked`
and `parseAssertLineLocked` detect task/state names longer than 15 characters.

### 5. `radio.config` parser refactor (`src/ams/mission_script_engine_parser_meta.cpp`)

`parseRadioConfigLineLocked` replaced `sscanf` with `TokenCursor` (new header
`src/ams/ams_token_cursor.h`) for key extraction and `parseFloatValue()` for
value parsing.  Benefits:

- Dotted parameter names (e.g. `monitor.alt.high`) parsed correctly.
- NaN, ±Inf, and values with trailing non-whitespace characters now rejected.
- `// NOLINT(readability-function-size)` added (clang-tidy suppression).

`parsePulseRequireContinuityLineLocked` — manual character loop replaced with
`TokenCursor::readIdentBounded` + `ensureEol()`.

`parsePulseMinAltLineLocked` — similarly refactored via `TokenCursor`.

**New file:** `src/ams/ams_token_cursor.h` — lightweight cursor utility used
by the parser refactor above.

### 6. `memset` → `std::fill_n` (`src/ams/mission_script_engine.cpp`)

`activate()`: `lastHkSlotMs_`, `lastLogSlotMs_` initialised with `std::fill_n`
instead of `memset` (correct type for `uint64_t[]`).

`deactivateLocked()`: `taskLastTickMs_` — same.

`resetSensorCachesLocked()`: `baroCacheTsMs_`, `baroCacheValid_`,
`gpsCacheTsMs_`, `gpsCacheValid_` — same.

### 7. Sensor non-finite guard (`src/ams/mission_script_engine_sensor.cpp`)

`readGpsFieldLocked`, `readBaroFieldLocked`, `readImuFieldLocked` — each
field-dispatch `switch` now falls through to a `std::isfinite(outVal)` check
after setting `outVal`.  On a non-finite result the sensor cache entry is
invalidated and the read returns `false`, preventing NaN/Inf values from
propagating into transition conditions or log CSV output.

`mission_script_engine_types.h` — `AliasEntry` doc comment updated to remove
the now-removed `[timeout=Nms]` option.

### 8. API layer — assert → graceful error (`src/api/flight/flight_handler.cpp`)

`handleMode`, `handleArm`, `handleAbort` — `ARES_ASSERT` on
`rawMode <= LAST` replaced with `LOG_E` + `sendError(client, 500, …) + return`.
The server no longer panics if an invalid mode value is ever present at a
system boundary.

### 9. Storage handler — assert → graceful error (`src/api/storage/storage_handler.cpp`)

Replaced six `ARES_ASSERT` calls with defensive checks and HTTP 500 returns:

- `handleLogsList`: `count > MAX_LOG_FILES` and JSON truncation.
- `sendFileChunked`: `bytesRead > CHUNK_SIZE` and `offset + bytesRead > fileSize`.
- `handleLogDeleteAll`: `count > MAX_LOG_FILES` and malformed entry name.
- `handleStorageHealth`: JSON truncation.

### 10. Radio dispatcher — CERT-1 rx-buffer guard (`src/comms/radio_dispatcher.cpp`)

`poll()` — added `received <= space` check.  If a driver violates its contract
and reports `received > space`, the rx buffer is reset to 0 with a `LOG_E`
rather than advancing `rxLen_` past `kRxBufLen` and causing an OOB read in
`processBuffer()`.

Redundant `ARES_ASSERT` range checks removed from the `ConfigParamId` loop
(loop bounds already enforce the range).

### 11. Boot hardening (`src/main.cpp`)

`setup()` — silent `(void)iface->begin()` calls replaced with explicit
`if (!iface->begin()) { LOG_W("BOOT", …); }` for baro, IMU, GPS, and pulse
subsystems.  `// NOLINT(readability-function-size)` added to suppress
clang-tidy warning on the function.

### 12. BMP280 driver (`src/drivers/baro/bmp280_driver.cpp`)

- `isfinite` → `std::isfinite` (MISRA-qualified call).
- Status-register read during NVM-load wait: failure now returns `false`
  immediately instead of silently continuing with `status = 0`.
- `ares_log.h` included; `TAG = "BMP280"` added for the new log call.

### 13. BMP280 test suite

**New directory:** `test/test_bmp280/` — new test suite exercising `begin()`
happy path, wrong chip ID rejection, I2C failure handling, `read()`,
`setSeaLevelPressure()`.

**`test/stubs/Wire.h`** — upgraded from a 2-method stub to a full virtual-method
`TwoWire` base class.  All methods are virtual so test suites can subclass and
script I2C responses.  `delay()` stub added (required by `bmp280_driver.cpp`).

**`platformio.ini`** — `bmp280_driver.cpp` and `test_bmp280` added to both
`sim` and `coverage` build environments.

### 14. `log_every` minimum interval raised to 100 ms

`AMS_LOG_INTERVAL_MIN_MS` effectively enforced at 100 ms (was 10 ms).
Affected scripts and tests updated:

- `sim_ams_scripts.h` — `kScriptLogHeaderRetry`, `kScriptVarFieldInLog`:
  `log_every 10ms` → `log_every 100ms`.
- `test_ams_flight_scenario.cpp` — `log_slot_header_retried_after_no_space`,
  `test_log_slot_row_has_crc8_field`: time advances updated 10 ms → 100 ms /
  20 ms → 200 ms.
- `test_ams_script_variants.cpp` — `test_var_field_in_log_report`: tick
  advance updated to 100 ms.

### 15. v4 strict-restore regression tests (`test/test_ams_integration/test_ams_script_variants.cpp`)

Four new tests covering `restoreCheckpointV4StrictLocked` boundary conditions:

- `test_checkpoint_v4_wrong_varcount_discarded` — checkpoint claims 2 vars
  but the script has 0; must be discarded (engine stays IDLE).
- `test_checkpoint_v4_wrong_hkcount_discarded` — checkpoint claims 0 HK
  slots but the loaded state has 1; must be discarded.
- `test_checkpoint_v4_truncated_confirm_section_discarded` — only 2 of 4
  confirm counters present; must be discarded.
- `test_checkpoint_v4_truncated_hold_section_discarded` — only 1 of 4
  hold/condElap pairs present; must be discarded.

Existing test vectors for `test_checkpoint_v4_confirm_restored`,
`test_checkpoint_millis_rollover_elapsed_wrap`, and
`test_checkpoint_resume_missing_file` updated to remove the `varCount` field
from the checkpoint body (varCount = 0 → no field written).

### 16. New parser unit tests (`test/test_ams_parser/`)

10 new tests added to `test_ams_parser_cond_and_meta.cpp` (suite grows from

### 17. New dispatcher tests (`test/test_dispatcher/`)
