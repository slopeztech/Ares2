# ARES v2.3.12 ŌĆö Changelog

**Released:** 2026-05-31
**Previous version:** 2.3.11
**Type:** Patch ŌĆö static initialisation order fix (P1-3); clang-tidy size warning resolved.

---

## Summary

This release closes the last C++ correctness finding from the post-v2.3.9 audit
and eliminates the remaining clang-tidy warning introduced by that fix.

**P1-3** ŌĆö `src/main.cpp` contained two objects with a cross-translation-unit
Static Initialisation Order Fiasco (SIOF) risk:

- `Bmp280Driver baro(Wire, ...)` ŌĆö `Wire` is `extern TwoWire Wire` defined in
  the Arduino framework's `Wire.cpp` (a different TU).  C++ does not guarantee
  that `Wire` is fully constructed before `baro`; the constructor stores only a
  reference and address (no methods called), so the practical risk was low, but
  the formal C++ guarantee was absent.

- `ApiServer apiServer(wifiAp, *kBaroIfaces[0], ...)` ŌĆö dereferences
  `kBaroIfaces[0]` which points to `baro`, compounding the dependency.

Both objects are now deferred to runtime:
- `baro`: constructed via **placement new** into a file-scope aligned buffer
  (`s_baroBuf`) immediately after `Wire.begin()` in `setup()`, where `Wire` is
  guaranteed initialised.  `kBaroIfaces[0]` and `kBaroDrivers[0].iface` are
  patched to `pBaro` at the same point.
- `apiServer`: constructed via placement new into `s_apiBuf` after
  `missionEngine.begin()` succeeds, factored out into the static helper
  `startApiServer()`.

`kBaroDrivers[]` is changed from `const` to non-const so `iface` can be
written once in `setup()` ŌĆö all downstream code only reads it.

A null guard is added at the top of `loop()`: if `setup()` returns before
`pApiServer` is constructed (storage failure, AMS init failure), `loop()`
idles via `vTaskDelay` instead of dereferencing a null pointer.  The TWDT
`esp_task_wdt_reset()` call is skipped on that path because `setup()` never
reached `esp_task_wdt_add()` in the early-exit branches.

**clang-tidy `readability-function-size`** ŌĆö the P1-3 additions pushed `setup()`
to 90 lines, exceeding the project threshold of 80.  The `ApiServer`
placement-new + `begin()` block is extracted into `static bool startApiServer()`
(18 lines), reducing `setup()` to 79 lines.  No semantic change.
