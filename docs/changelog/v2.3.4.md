# ARES v2.3.4 — Changelog

**Released:** 2026-05-26
**Previous version:** 2.3.3
**Type:** Patch — security: rolling timestamp anti-replay window for COMMAND frames ([C1]).

---

## Summary

This release closes the replay-attack residual left open after v2.3.3.  The
HMAC-SHA256 tag introduced in v2.3.3 prevents *forgery*, but a valid captured
frame could still be replayed after a device reboot (the 64-entry SeqBitmap is
volatile and resets to zero at power-on).

The fix adds a **rolling timestamp window** of ± 5 s (`CMD_TIMESTAMP_WINDOW_MS`)
to every authenticated COMMAND frame.  The ground-station tool must stamp each
frame with the current uptime (in ms) in the new `CommandHeader.timestampMs`
field.  The dispatcher rejects any frame whose timestamp lies outside the
acceptance window — defeating cross-boot replays that the SeqBitmap alone
cannot prevent.

As a housekeeping improvement, the MAC + timestamp validation logic is extracted
from `handleCommand()` into a new private helper `checkCommandMac()`, bringing
the function's line count within the clang-tidy `readability-function-size`
threshold (80 lines).

## Changes

### `src/comms/ares_radio_protocol.h`

| Location | Change |
|----------|--------|
| `struct CommandHeader` | Added `uint32_t timestampMs` field at bytes [2..5] (little-endian); total size 2 → **6 bytes** |
| `static_assert` | Added `static_assert(sizeof(CommandHeader) == 6)` |
| New constant | `constexpr uint32_t CMD_TIMESTAMP_WINDOW_MS = 5000U` — acceptance window for timestamp anti-replay |

### `src/comms/radio_dispatcher.h`

| Location | Change |
|----------|--------|
| New private method | `bool checkCommandMac(const proto::Frame& frame, uint32_t nowMs)` — authenticates and validates timestamp; sends NACK on any failure; returns bool |

### `src/comms/radio_dispatcher.cpp`

| Location | Change |
|----------|--------|
| `checkCommandMac()` *(new)* | Verifies `FLAG_MAC` presence, HMAC-SHA256 tag (constant-time), and rolling timestamp window; sends `NACK(HMAC_INVALID)` and returns `false` on any failure |
| `handleCommand()` | Delegates MAC+timestamp checks to `checkCommandMac()`; reduced from 101 to ~50 lines (clang-tidy `readability-function-size` resolved) |
| `executeCommand()` payload offsets | All byte offsets shifted +4 to account for the expanded `CommandHeader` (bytes [2..5] now occupied by `timestampMs`): `SET_MODE`, `SET_FCS_ACTIVE`, `SET_TELEM_INTERVAL`, `SET_CONFIG_PARAM` |
