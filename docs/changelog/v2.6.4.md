# Changelog — v2.6.4

**Date:** 2026-06-22  
**Type:** Refactoring / Architecture (Minor)

---

## Summary

Version 2.6.4 abstracts SERIAL.report output through a hardware-agnostic `SerialInterface`
interface, eliminating direct Arduino dependencies in the AMS telemetry layer and enabling
portable, testable serial output. The change maintains full backward compatibility while
improving future maintainability: switching to a different serial transport (e.g., CDC, ESP-IDF UART,
or test stub) requires only a new driver implementation, not engine changes.

**Key achievements:**
- ✅ No `#ifdef ARDUINO` or `Serial.` calls in core AMS engine.
- ✅ Scalable registration pattern matching GPS/BARO/COM/IMU drivers.
- ✅ MISRA/CERT compliant with safe null-pointer handling.
- ✅ Full backward compatibility—no API changes visible to scripts or users.

## Architecture Changes

### 1. New HAL Interface: `SerialInterface`

**File:** `src/hal/serial/serial_interface.h`

Introduced a minimal pure virtual interface for serial output:

```cpp
class SerialInterface {
public:
    virtual ~SerialInterface() = default;
    
    /// Query available TX buffer space (bytes).
    virtual uint32_t availableForWrite() const = 0;
    
    /// Write raw byte block; returns bytes accepted.
    virtual uint32_t write(const uint8_t* data, uint32_t len) = 0;
};
```

**Design rationale:**
- Matches pattern established by `GpsInterface`, `BarometerInterface`, etc.
- Minimal API: only two methods needed for bounded, backpressure-aware telemetry output.
- Pure virtual guarantees link-time satisfaction of contract.

### 2. Arduino Adapter: `ArduinoSerialInterface`

**Files:**
- `src/drivers/serial/arduino_serial_interface.h`
- `src/drivers/serial/arduino_serial_interface.cpp`

Concrete implementation wrapping Arduino `Print` (e.g., `Serial` global):

```cpp
class ArduinoSerialInterface : public SerialInterface {
public:
    explicit ArduinoSerialInterface(Print& serial);
    uint32_t availableForWrite() const override;
    uint32_t write(const uint8_t* data, uint32_t len) override;
};
```

**Implementation notes:**
- Captures Arduino `Print&` by reference (no allocation, no ownership).
- Safe cast from signed (`int`) to unsigned (`uint32_t`) with bounds checking.
- No includes of `<Arduino.h>` needed outside this translation unit.

### 3. MissionScriptEngine Integration

**Files:** `src/ams/mission_script_engine.h`, `src/ams/mission_script_engine.cpp`

#### Header Changes

Added public registration method:
```cpp
void setSerialInterface(SerialInterface* serialIface);
```

Added member:
```cpp
SerialInterface* serialIface_ = nullptr;  ///< Nullable — SERIAL.report sink
```

Added private backpressure helpers:
```cpp
bool canWriteSerialReportLineLocked(size_t bytesNeeded) const;
bool writeSerialReportBytesLocked(const uint8_t* data, size_t len) const;
```

#### Telemetry Layer Refactoring

**File:** `src/ams/mission_script_engine_telemetry.cpp`

**Removed:**
```cpp
static bool canWriteSerialReportLine(size_t bytesNeeded) {
#ifdef ARDUINO
    // ... Serial.availableForWrite() logic
#endif
}
```

**Replaced with:**
```cpp
bool MissionScriptEngine::canWriteSerialReportLineLocked(size_t bytesNeeded) const {
    if (serialIface_ == nullptr) return false;
    return serialIface_->availableForWrite() >= bytesNeeded;
}

bool MissionScriptEngine::writeSerialReportBytesLocked(...) const {
    if (serialIface_ == nullptr) return false;
    return serialIface_->write(data, len) == len;
}
```

**Effect:** Telemetry tick now gracefully skips SERIAL.report if no sink registered (nullable
by design), maintains silent backpressure drop policy under congestion.

### 4. Hardware Registry Extension

**Files:**
- `src/sys/hardware/installed_driver_registry.h`
- `src/sys/hardware/installed_driver_registry.cpp`

Extended central driver registry to support serial drivers:

| Change | Details |
|--------|---------|
| `DriverKind::SERIAL_IO` | New enum (named `SERIAL_IO` not `SERIAL` to avoid collision with Arduino macro `#define SERIAL 0x0`). |
| `SerialDriverEntry` | New typedef `DriverEntry<SerialInterface>`. |
| `serialDrivers()` | New accessor returning registered serial drivers. |
| `defaultSerialDriverModel()` | Returns "SERIAL0" or "NONE". |
| `InstalledHardwareRefs.serialOut` | New field for binding serial interface. |
| `bindInstalledHardware()` | Updated signature to accept `SerialInterface*`. |
| `isSupportedModel()` | Updated switch to include `SERIAL_IO` case. |

**Registry entry:**
```cpp
SerialDriverEntry g_serialEntries[] = {
    { "SERIAL0", nullptr }  // Model name and interface pointer (bound at init)

```

### 6. Platform Bootstrap Wiring

**File:** `src/main.cpp`

#### Static Instance
```cpp
static ArduinoSerialInterface serialOut(Serial);
```

#### Binding
```cpp
static void bindInstalledDrivers(BarometerInterface* barometer,
                                 SerialInterface* serialOut) {
    const ares::hardware::InstalledHardwareRefs refs = {
        barometer, &gps, &radio, &imu, &imu2, serialOut
    };
    ares::hardware::bindInstalledHardware(refs);
}

// Called in setup():
bindInstalledDrivers(pBaro, &serialOut);
```

#### Resolution
```cpp
static SerialInterface* resolveSerialOutput() {
    const auto& serials = ares::hardware::serialDrivers();
    if (serials.count == 0U || serials.entries == nullptr) return nullptr;
    return serials.entries[0U].iface;
}

// Called in setup() after registry binding:
missionEngine.setSerialInterface(resolveSerialOutput());
```

**Effect:** MissionScriptEngine consumes serial sink from central registry, not from direct
class reference, enabling future transport swaps without touching engine code.

## Migration Guide

### For Platform Developers

If you implement a custom serial driver (e.g., for a different transport), follow this pattern:

1. **Implement `SerialInterface`:**
   ```cpp
   class MyCustomSerialDriver : public SerialInterface {
   public:
       uint32_t availableForWrite() const override { /* ... */ }
       uint32_t write(const uint8_t* data, uint32_t len) override { /* ... */ }
   };
   ```

2. **Register in `main.cpp`:**
   ```cpp
   static MyCustomSerialDriver mySerial;
   
   // In bindInstalledDrivers():
   bindInstalledDrivers(pBaro, &mySerial);
   ```

3. **Update registry (if multi-driver support needed):**
   - Add entry to `g_serialEntries[]` in `installed_driver_registry.cpp`.
   - Update `serialDrivers().count` accordingly.

### For AMS Script Authors

No changes required. SERIAL.report continues to work as documented:
- Output flows to registered serial sink.
- Under TX backpressure, samples are silently dropped to maintain tick latency.
