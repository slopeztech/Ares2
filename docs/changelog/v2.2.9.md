# ARES v2.2.9 — Changelog

**Released:** 2026-05-19  
**Previous version:** 2.2.8  
**Type:** Patch — security fix: replace equality-only SEQ duplicate check with 64-slot sliding-window anti-replay bitmap ([H5]).

---

## Summary

Resolves security finding **[H5]**: `proto::isDuplicate(seq, lastSeq)` compared a
new frame's SEQ byte only against the *last accepted* SEQ.  An attacker could
replay any previously accepted command by sending it with a new (different) SEQ,
e.g. SEQ N−1 immediately after SEQ N was accepted.

The fix introduces `proto::SeqBitmap` — a 64-slot sliding-window bitmap that
tracks the last 64 accepted SEQ values.  A frame is rejected if:
- its SEQ matches the current highest accepted SEQ (exact duplicate), or
- its SEQ falls within the 63-position look-back window and the corresponding
  bit is already set, or
- its SEQ is more than 63 positions behind the current highest (outside window
  → too old to distinguish, treated as replay).

`isDuplicate()` is kept unchanged (existing tests still pass); `SeqBitmap` is
the authoritative mechanism used by `RadioDispatcher`.

## Changes

### `src/comms/ares_radio_protocol.h`

| Location | Change |
|----------|--------|
| After `isDuplicate()` declaration | Added `SeqBitmap` struct with `checkAndMark(uint8_t seq) -> bool` and `kWindowSize = 64` |
| `isDuplicate()` Doxygen | Added note referencing `SeqBitmap` for full sliding-window protection |

### `src/comms/radio_dispatcher.h`

| Location | Change |
|----------|--------|
| Private fields | Removed `uint8_t lastRxSeq_` and `bool seenFirst_`; added `proto::SeqBitmap rxSeqBitmap_` |

### `src/comms/radio_dispatcher.cpp` — `handleCommand()`

| Location | Change |
|----------|--------|
| Duplicate guard (APUS-4.7) | Replaced `seenFirst_ && isDuplicate(frame.seq, lastRxSeq_)` with `rxSeqBitmap_.checkAndMark(frame.seq)` |
| After acceptance ACK | Removed `lastRxSeq_ = frame.seq; seenFirst_ = true;` (state now managed by `SeqBitmap`) |
| Log message | Updated to `"duplicate/replay — discarded"` |

### `test/test_radio_protocol/test_misc.cpp` — 8 new `SeqBitmap` tests

| Test | Scenario |
|------|----------|
| `test_seq_bitmap_first_seq_not_duplicate` | First SEQ ever is never a duplicate |
| `test_seq_bitmap_same_seq_is_duplicate` | Same SEQ submitted twice → duplicate |
| `test_seq_bitmap_sequential_no_duplicate` | 64 sequential SEQs, none duplicated |
| `test_seq_bitmap_outside_window_rejected` | SEQ 64 behind current high → outside window |
| `test_seq_bitmap_back_edge_of_window_accepted` | SEQ 63 behind current high → inside window, accepted once |
| `test_seq_bitmap_out_of_order_within_window` | Late-arriving SEQ accepted once, replay rejected |
| `test_seq_bitmap_wraparound_255_to_0` | SEQ wraps 255→0 correctly |
| `test_seq_bitmap_jump_ahead_resets_window` | Jump >64 ahead causes full window reset |

### `test/test_dispatcher/test_command_dispatch.cpp` — 2 new anti-replay dispatcher tests

| Test | Scenario |
|------|----------|
| `test_cmd_replay_previous_seq_rejected` | SEQ N−1 replayed after SEQ N accepted → discarded |
| `test_cmd_replay_outside_window_rejected` | SEQ 64 behind current high → discarded |
