# Changelog — v2.1.2

**Date:** 2026-05-17
**Type:** Improvement / Limit Increase (Patch)

---

## Summary

Version 2.1.2 raises the AMS state machine capacity from 10 to 16 states and
widens the internal BFS/DFS reachability bitmask from `uint16_t` to `uint32_t`,
pushing the hard architectural ceiling from 16 to 32.  The immediate practical
benefit is support for realistic multi-phase flight profiles (e.g. 13-state
PRE_FLIGHT → … → EMG_RECOVERY) that were impossible under the old limit of 10.

1. **`AMS_MAX_STATES` 10 → 16** — parser now accepts up to 16 states per script.
2. **BFS/DFS bitmask `uint16_t` → `uint32_t`** — hard architectural ceiling
   raised from 16 to 32; `static_assert` updated accordingly.
3. **Test suite updated** — two existing tests fixed for the new limit;
   two new tests added verifying the 13-state flight profile and large-graph
   BFS correctness.

---

## Improvements

### `AMS_MAX_STATES` raised from 10 to 16

**File:** `src/config.h`

`AMS_MAX_STATES` has been increased from `10` to `16`.  This allows realistic
multi-phase mission scripts (e.g. 13 states for a full rocket flight) to be
loaded and validated without hitting the parser's state-count ceiling.

The `///< ...` Doxygen comment on the constant has been updated to reference
`uint32_t` and the new hard ceiling of 32.

### BFS/DFS bitmask widened to `uint32_t`

**Files:** `src/ams/mission_script_engine_parser_finalize.cpp`,
`src/ams/mission_script_engine.h`

The BFS reachability algorithm in `validateAssertionsLocked` used a `uint16_t`
bitmask (one bit per state index) and a matching `static_assert` that enforced
`AMS_MAX_STATES ≤ 16`.  With `AMS_MAX_STATES` now set to exactly 16, there was
no architectural headroom.

All bitmask variables and casts have been widened to `uint32_t`:

| Location | Change |
|---|---|
| `computeBfsReachabilityLocked` parameter | `uint16_t&` → `uint32_t&` |
| BFS seed and visit `static_cast` (×3) | `uint16_t` → `uint32_t` |
| `DfsFrame::pathMask` field type | `uint16_t` → `uint32_t` |
| DFS initial frame seed `static_cast` | `uint16_t` → `uint32_t` |
| DFS cycle-check `static_cast` | `uint16_t` → `uint32_t` |
| DFS frame-push path-union `static_cast` (×2) | `uint16_t` → `uint32_t` |
| `evaluateOneAssertionLocked` parameter | `uint16_t` → `uint32_t` |
| `allMask` type and `static_cast` in `no_dead_states` | `uint16_t` → `uint32_t` |
| `reachable` local in `validateAssertionsLocked` | `uint16_t` → `uint32_t` |
| `static_assert` ceiling | `<= 16U` / `"…<= 16"` → `<= 32U` / `"…<= 32"` |

The Doxygen comment above `computeBfsReachabilityLocked` has been updated to
reflect the new 32-state ceiling.
