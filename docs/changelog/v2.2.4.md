**Date:** 2026-05-18  
**Type:** Patch  
**Branch:** `feature/h2-log-coverage`

## Summary

Adds `test_api_routing`, a new PlatformIO native test environment that exercises
the `ApiServer::routeRequest()` path end-to-end using a `SimWiFiClient` mock.
Ten tests cover HTTP method routing, CORS pre-flight, error path responses, and
bearer-token authentication.

### New PlatformIO environment

```ini
[env:api_routing]
platform = native
build_flags = -std=c++17 ... -DARES_NDEBUG -DARES_SITL
test_filter = test_api_routing
```

`ARES_NDEBUG` is required to suppress the RTOS task-handle assertion in
`handleClient()` which is meaningless in a native test process.  Tests call
`routeRequest()` directly, bypassing `handleClient()` entirely.
