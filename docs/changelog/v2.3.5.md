# ARES v2.3.5 — Changelog

**Released:** 2026-05-27
**Previous version:** 2.3.4
**Type:** Patch — minor defect fixes: documentation clarity (BUG-13), CSV robustness (BUG-15), compile-time sentinel guard (BUG-16).

---

## Summary

This release addresses three minor defects identified in the post-v2.3.4 audit.
No functional behaviour is changed; existing firmware images are unaffected.

- **BUG-13** — `loop()` comment now documents the `pdMS_TO_TICKS` tick-resolution
  truncation and the effective `TickType_t` overflow limit at the default 1000 Hz
  tick rate, preventing silent saturation surprises when the tick rate is changed.

- **BUG-15** — `HkField::label` values are sanitised before being written as CSV
  column headers.  Any `,` or `"` characters are replaced with `_`, preventing
  malformed CSV output if a user-supplied label contains those reserved characters.
  The `HkField::label` documentation now states this rule explicitly.

- **BUG-16** — Two `static_assert` guards have been added immediately after the
  `EngineStatus` enum definition.  They verify that `FIRST` aliases `IDLE` (0) and
  `LAST` aliases `LOADED` (4) at compile time, so inserting a new enumerator outside
  the current range will produce a build error rather than silently breaking all
  range-validation loops.

## Changes

### `src/main.cpp`

| Location | Change |
|----------|--------|
| `loop()` — `vTaskDelay` call | Added comment: tick resolution = `1000/configTICK_RATE_HZ` ms/tick; `TickType_t` (uint32_t) overflow at ~49.7 days @ 1000 Hz, well above `LOOP_SLEEP_MAX_MS` |

### `src/ams/mission_script_engine_types.h`

| Location | Change |
|----------|--------|
| `HkField::label` doc comment | Added note: must not contain `,` or `"`; such characters are replaced with `_` at CSV write time |
| `EngineStatus` — after closing `}` | Added `static_assert(FIRST == 0U)` and `static_assert(LAST == 4U)` with descriptive messages (BUG-16 / CERT-6.1) |

### `src/ams/mission_script_engine_telemetry.cpp`

| Location | Change |
|----------|--------|
| New `static sanitiseCsvLabel()` helper | Copies a label to a caller-supplied buffer, replacing `,` and `"` with `_`; placed before `appendLogReportLocked` |
| `writeLogHeaderIfNeededLocked()` — header loop | Calls `sanitiseCsvLabel()` before each `snprintf(",%s", label)` for the legacy single-slot path (`logFields[]`) |
| `appendLogReportSlotLocked()` — header loop | Calls `sanitiseCsvLabel()` before each `snprintf(",%s", label)` for the slot-based path (`slot.fields[]`) |
