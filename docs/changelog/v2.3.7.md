# ARES v2.3.7 ŌĆö Changelog

**Released:** 2026-05-28
**Previous version:** 2.3.6
**Type:** Patch ŌĆö security hardening: global mission-upload buffer ownership invariant enforced at the access point (BUG-3).

---

## Summary

This release closes a latent data-race risk identified in the post-v2.3.6 audit.

**BUG-3** ŌĆö `g_missionUploadBuf` (4 KB static buffer for AMS script uploads) had its
ownership invariant enforced only by a single `ARES_ASSERT` at the top of
`handleClient()`, detached from the actual buffer access sites.  Any refactor that
moved the mission-upload handler to a second task would introduce a data race with
no compile-time or link-time detection.

The fix encapsulates the buffer in a non-instantiable `struct MissionUploadBuf`
with a static `get(TaskHandle_t expected)` accessor.  The `ARES_ASSERT` now lives
inside `get()`, so it is impossible to access the buffer in **debug builds**
without the ownership check executing.  In release builds (`-DARES_NDEBUG`)
`ARES_ASSERT` is a no-op ŌĆö the architectural invariant (single-task access) must
be maintained by design; the encapsulation and the unmissable Doxygen block make
any violation immediately apparent during code review and debug testing.  An
unmissable Doxygen block documents the invariant directly above the struct
definition.

## Detailed description

### BUG-3: `MissionUploadBuf` singleton (`src/api/api_server.cpp`)

**Before:**

```cpp
// (fragile comment only)
static char g_missionUploadBuf[ares::AMS_MAX_SCRIPT_BYTES + 1U] = {};

void ApiServer::handleClient(WiFiClient& client)
{
    // Guard detached from the buffer ŌĆö could be removed without noticing.
    ARES_ASSERT(xTaskGetCurrentTaskHandle() == taskHandle_);
    ...
    char* body = isMissionUpload ? g_missionUploadBuf : smallBody;
}
```

**After:**

```cpp
/**
 * OWNERSHIP INVARIANT: the API task is the sole accessor of this buffer.
 * get() asserts xTaskGetCurrentTaskHandle() == expected at every call site.
 */
struct MissionUploadBuf
{
    // All constructors/destructor = delete  (CERT-18.3)
    static char* get(TaskHandle_t expected)
    {
        ARES_ASSERT(xTaskGetCurrentTaskHandle() == expected);
        return buf_;
    }
    static constexpr size_t SIZE = ares::AMS_MAX_SCRIPT_BYTES + 1U;
private:
    static char buf_[SIZE];
};

void ApiServer::handleClient(WiFiClient& client)
{
    ...
    if (isMissionUpload)
    {
        body = MissionUploadBuf::get(taskHandle_);  // assert here
        memset(body, 0, MissionUploadBuf::SIZE);
    }
}
```

**Properties of the new design:**

- The assert is co-located with the buffer access ŌĆö impossible to bypass accidentally.
- The `OWNERSHIP INVARIANT` comment is unmissable (Doxygen block at definition site).
- `buf_` is `private` ŌĆö no external code can alias or take its address.
- `SIZE` is a canonical `constexpr` ŌĆö no `sizeof(g_missionUploadBuf)` magic elsewhere.
- All six copy/move/destructor specials are `= delete` (CERT-18.3,
  `cppcoreguidelines-special-member-functions`).
- BSS footprint and runtime behaviour are identical to the original.
