# ARES v2.6.1 - Changelog

Previous version: 2.6.0
Type: Minor - AMS SERIAL.report support in log_every, with parser/runtime/docs updates.

---

## Summary

This release adds a new AMS reporting directive for live serial monitoring:

- Added SERIAL.report as a valid report block inside log_every cadence slots.
- Each log_every slot can now target one output:
  - LOG.report -> CSV row append to mission log file in LittleFS.
  - SERIAL.report -> formatted line to serial monitor.
- Kept field grammar identical between LOG.report and SERIAL.report (label: ALIAS.field or label: varname).
- Extended offline validator (scripts/ams_parser.py) to accept SERIAL.report and validate its placement.
- Updated AMS documentation (programming guide + standard) to describe syntax and behavior.
- Added `RUNTIME.*` execution data fields for reusable runtime observability across AMS.

---

## Language and runtime changes

1. New AMS directive

```ams
log_every 250ms:
  SERIAL.report {
    altitude: BARO.alt
    reference: zero_alt
  }
```

Rules:
- SERIAL.report requires a preceding log_every Nms: block.
- SERIAL.report uses the same field parser as LOG.report.
- Cadence scheduling and starvation guards are unchanged.

2. Slot output selection

A log_every slot now stores its output target:
- false: CSV file output (LOG.report)
- true: serial output (SERIAL.report)

During tick dispatch, due log slots route accordingly.

3. Runtime execution data alias

Added reserved virtual alias `RUNTIME` with numeric fields:

- `RUNTIME.uptime_ms`
- `RUNTIME.state_idx`
- `RUNTIME.state_elapsed_ms`
- `RUNTIME.script_elapsed_ms`
- `RUNTIME.status_bits`
- `RUNTIME.engine_status`
- `RUNTIME.exec_enabled`

Usage scope:

- Report fields (`HK.report`, `LOG.report`, `SERIAL.report`)
- Guards (`conditions:`)
- Transition conditions (including `delta`, `falling`, `rising` forms)
- `set` expression terms

HK payload compatibility:

- Directly mapped to fixed binary payload slots when available:
  - `RUNTIME.uptime_ms` -> `timestampMs`
  - `RUNTIME.state_idx` -> `flightPhase`
  - `RUNTIME.status_bits` -> `statusBits`
- Other runtime fields remain available for LOG/SERIAL and conditions without
  modifying the fixed 78-byte APUS TelemetryPayload layout.
