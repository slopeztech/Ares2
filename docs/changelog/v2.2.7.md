# ARES v2.2.7 — Changelog

**Released:** 2026-05-19  
**Previous version:** 2.2.6  
**Type:** Patch — refactor: split monolithic `mission_script_engine.h` into focused headers ([M1]).

---

## Summary

Resolves todo item **[M1]**: `mission_script_engine.h` had grown to 1 438 lines and
contained more than 40 nested struct/enum definitions, harming compile times and
coupling unrelated concerns together.  The definitions have been extracted into two
new focused headers; the main header now holds only the `MissionScriptEngine` class
declaration (~780 lines).  No runtime behaviour changes; all 137 native unit tests
pass unchanged.

---

## Changes

New header containing all **parser data-model type definitions** that were formerly
private-nested inside `MissionScriptEngine`.  All types are now at `ares::ams`
namespace level, which removes a spurious compilation dependency on the class
declaration for translation units that only need the type definitions.

### `src/ams/mission_script_engine.h` *(modified)*

- Added `#include "ams/mission_script_engine_internal.h"` and
  `#include "ams/mission_script_engine_types.h"` after the existing HAL includes.
- Removed all extracted type definitions (~780 lines removed).
- Short reference comments added at each removed section pointing to the new headers.
- **Member variable declarations unchanged** — `pendingCheckpoint_`,
  `pendingAppends_[kMaxPendingAppends]`, `monitorSlots_[kMaxMonitorSlots]`, etc.
  remain in the class body and resolve correctly via `ares::ams` namespace.
- **Size:** 78 071 chars → 47 468 chars (−39%).
