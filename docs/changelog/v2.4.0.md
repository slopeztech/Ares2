# ARES v2.4.0 ‚Äî Changelog

**Released:** 2026-06-02
**Previous version:** 2.3.15
**Type:** Minor ‚Äî new AMS `every ‚Ä¶ via ALIAS:` syntax for per-slot COM routing.

## Summary

This release adds **A2-3: multi-radio HK routing via alias**.  A new optional
`via ALIAS` clause on `every Nms` blocks allows each HK slot to target a
specific COM peripheral instead of always using the primary radio.  The
feature is backward-compatible: scripts without `via` are unaffected.

Key highlights:

- **New AMS syntax** ‚Äî `every Nms via ALIAS:` routes the HK slot's telemetry
  frames through the COM peripheral bound to `ALIAS`.
- **`HkSlot.comAlias` field** ‚Äî 16-byte alias name stored per slot;
  empty = use `primaryCom_` (unchanged behaviour).
- **Runtime fallback warning** ‚Äî if the alias cannot be resolved at dispatch
  time a `LOG_W` is emitted and the primary COM is used instead of failing
  silently.
- **Parser validation** ‚Äî alias must be declared with `include ‚Ä¶ as ALIAS`,
  must be of kind `COM`, and must not produce trailing garbage.
- **8 new tests** ‚Äî 5 parser edge-case tests (unknown alias, below-min
  interval, trailing garbage, empty alias, two-slot mixed) and 3 runtime
  integration tests using `DualRadioFixture`.

After this release the master CI pipeline continues to report

## Changes

### 1. `HkSlot` new field (`src/ams/mission_script_engine_types.h`)

`HkSlot` gains a `char comAlias[16]` field (zero-initialised):

```cpp
struct HkSlot {
    uint32_t everyMs      = 0U;
    uint8_t  fieldCount   = 0U;
    char     comAlias[16] = {};   // A2-3: target COM alias; empty = use primaryCom_
    HkField  fields[ares::AMS_MAX_HK_FIELDS] = {};
};
```

The field is serialised to zero by default, so existing checkpoints are
unaffected.

### 2. Parser ‚Äî `every ‚Ä¶ via ALIAS:` (`src/ams/mission_script_engine_parser_state.cpp`)

`parseEveryLineLocked` gains a `via` branch:

- Detects `"ms via "` substring after the numeric interval.
- Extracts the alias name and validates length (1‚Äì15 characters).
- Looks up the alias via `findAliasLocked`; rejects if not found or not of
  kind `PeripheralKind::COM`.
- Requires a colon immediately after the alias name; rejects trailing garbage.
- Stores the validated alias name into `st.hkSlots[slotIdx].comAlias`.

Error messages produced:

| Condition | Message |
|-----------|---------|
| Alias not declared or not COM | `"every via: alias is not a COM include"` |
| Alias name empty or too long | `"every via: alias name empty or too long"` |
| Trailing characters after `':'` | `"every via: unexpected characters after ':'"` |
| Missing `':'` after alias name | `"every via: missing ':'"` |

### 3. Telemetry dispatch (`src/ams/mission_script_engine_telemetry.cpp`)

`sendHkReportSlotLocked` ‚Äî resolves `slot.comAlias` at dispatch time:

```cpp
RadioInterface* via = nullptr;
if (slot.comAlias[0] != '\0')
{
    const AliasEntry* ae = findAliasLocked(slot.comAlias);
    if (ae != nullptr && ae->kind == PeripheralKind::COM && ae->driverIdx < comCount_)
    {
        via = comDrivers_[ae->driverIdx].iface;
    }
    else
    {
        LOG_W(TAG, "HK slot: via alias '%s' unresolved ‚Äî using primaryCom_", slot.comAlias);
    }
}
transmitHkFrameLocked(tm, "HK slot", via);
```

`sendFrameLocked` doxygen updated: "via @p com if non-null, otherwise via
`primaryCom_`" (was: "via the primary COM interface").
